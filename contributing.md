# Contributing

Thanks for helping grow this list. A few guidelines keep it useful.

## What belongs here

- **Open data.** The dataset must be downloadable or streamable without a paywall, application gate, or restrictive license. If access requires a free registration, note that in the description.
- **Real data, not just code.** Analysis libraries and pipelines do not belong in the dataset sections. Formats, standards, and toolkits the data depends on go under **Standards & Tools**.
- **Brain / nervous-system data.** Any species, any modality.

## Entry format

Add entries as a single bullet, following the [Awesome](https://github.com/sindresorhus/awesome) convention:

```markdown
- [Name](https://link) - Description starting with a capital and ending with a period.
```

- Put the entry under the most specific category that fits. If the right category does not exist yet, add it (and update the Contents table of contents).
- Prefer the canonical landing page over a deep link, but link the actual data archive when that is the useful destination.
- Keep descriptions to a sentence or two: what it is, species, modality, and where the data lives.
- Sort entries within a category alphabetically unless there is a reason not to.

## Submitting

1. Fork the repo and create a branch.
2. Make your change. Check that links resolve and that the table of contents still matches the headings.
3. Open a pull request describing what you added and why it is openly accessible.
